HowTo - use tkbibtex under Windows NT

Settings

After a little confusion I now run tkbibtex successfully on Windows NT 4.0 SP5. Both Tcl/Tk and tkbibtex are installed in separate network folders on our server. Users have read-only access to these folders. Tcl/Tk is version 8.3.0, tkbibtex is version 7.

Ways it didn't work

Somehow you got to open tkbibtex with a bib file. Trying to open tkbibtex first and open the bib file from there won't work (although this is what happens when you try to assign the bib file type in Explorer). So the following attempts are not successful:

  • Opening wish and trying to open the source tkbibtex.tcl
  • Trying to start tkbibtex.tcl alone from the command prompt using tclsh83
  • Trying to start tkbibtex.tcl alone from the command prompt using wish83

Requirements

  • tkbibtex

    • rename it to tkbibtex.tcl after downloading
  • Tcl/Tk

Get it to run

Install Tcl/Tk in the directory of your liking. The same is true for tkbibtex.

Running it through a batch file

This is my favourite method as it works and Explorer doesn't have to handle yet another extension.

Create a batch file for every bib file (not a big deal indeed as long as Tcl/Tk and tkbibtex stay in their folders). Give it the same name and place it in the same folder as the bib file, so in Explorer it resides at the side of the bib file. A doubleclick on the batch file and your bib file opens in tkbibtex.

--- 8< --- cut here ---

rem +-------------------------------------------------------------+
rem | foo.bat - sample batch file for opening foo.bib in tkbibtex |
rem +-------------------------------------------------------------+

path_to_wish\wish83 path_to_tkbibtex\tkbibtex.tcl foo.bib

--- 8< --- cut here ---

Running it through Explorer

As I'm running a german version of WinNT not all names of dialogs and fields may be correct. Feel free to tell me their real (english) names.

Another option is to define the file extension .bib in Explorer. To do this open Explorer, click on View | Options | File Types and check if there is something defined for .bib files (we don't want to mess around). If there is no entry click on "New Type" and enter the following values:

  • Description: BiBTeX Document
  • Mime Type: none (leave the field empty)
  • Actions:

    • First create a new entry (click on "New")
    • Name: open
    • Executable for this ...: path_to_wish\wish83.exe path_to_tkbibtex\tkbibtex.tcl "%1"

      • This calls wish with tkbibtex.tcl and the file name as parameters
  • click "Ok"
  • click "Ok"

Now you can doubleclick on you bib file in Explorer and tkbibtex will open. Unluckily the bib file is not opened with tkbibtex. Seemingly the file name gets lost on its way to wish. If somebody comes up with a smarter method of doing this please let me know.
